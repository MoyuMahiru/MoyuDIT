# 架构设计

## 1. 产品定位

Photo Ingest Studio 不是图库管理软件，而是导入工具。它负责把素材安全地从储存卡导入到工作盘，并尽量减少摄影师手工整理目录和重复校验的时间。

## 2. 用户主流程

1. 插入储存卡或手动选择来源目录
2. 软件分析图片并读取 EXIF
3. 给出本次导入概览：文件数、日期范围、机型、格式
4. 用户确认导入方案：目标目录、归档方式、是否按已有星级筛选
5. 开始拷贝
6. 拷贝完成后做哈希比对
7. 记录导入历史，支持重试失败文件

## 3. 模块划分

### 3.1 前端

- 来源页：选择卡或目录，触发素材分析
- 分析页：显示文件数、日期范围、机型、体积
- 方案页：目标根目录、归档方式、重复策略、校验方式
- 执行页：导入进度、成功/跳过/失败
- 历史页：过去任务、重复导入、失败重试

### 3.2 Rust 后端

- `device`：枚举移动卷和用户选择目录
- `scanner`：扫描文件并过滤格式
- `metadata`：优先通过 ExifTool 读取拍摄时间、机型、评级；缺失时回退到文件修改时间
- `planner`：根据模板计算目标路径
- `ingest`：负责拷贝、重命名、临时文件和校验
- `hash`：MD5 和 BLAKE3
- `history`：SQLite 记录任务结果

## 4. 路径模板建议

建议支持以下变量：

- `{YYYY}`
- `{MM}`
- `{DD}`
- `{YYYY-MM-DD}`
- `{camera}`
- `{lens}`
- `{rating}`

示例：

```text
{YYYY}/{YYYY-MM-DD}/{camera}
```

输出：

```text
2026/2026-03-26/Canon EOS R5
```

## 5. 导入与校验策略

### 5.1 安全拷贝

每个文件按这个流程处理：

1. 打开源文件
2. 创建目标临时文件，例如 `filename.ext.part`
3. 读取源文件块并写入目标
4. 同时计算源文件哈希
5. 写入完成后计算目标文件哈希
6. 两者一致才改名为正式文件
7. 记录数据库

### 5.2 去重

去重条件建议至少包括：

- 文件大小
- 哈希值

策略：

- `skip`：已存在则跳过
- `rename`：目标文件自动追加序号
- `keep-both`：保留两份，并记录关联关系

## 6. 星级设计

MVP 只做一层：

- 读取已有评级，并把它作为导入过滤条件

后续再补：

- 写入 `XMP sidecar`
- 软件内人工打星
- 评级回写策略

## 7. 第一个可交付版本

MVP 应该只做下面这些：

- 选择源目录或储存卡
- 扫描 JPG 和主流 RAW
- 读取拍摄时间
- 按日期或日期 + 机型生成目标目录
- 只导入已有评级照片或全部导入
- 执行导入并做 MD5 校验
- 显示成功 / 跳过 / 失败统计

先把安全导入链路做扎实，再补星级、缩略图和历史记录。
